## 2. 首次配置

### 2.1 配置 LLM API

编辑项目根目录下的 `.env` 文件：

```env
SILICONFLOW_API_KEY=sk-你的密钥
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
LLM_MODEL=deepseek-ai/DeepSeek-V4-Flash

# 如需走本地代理（不需要时注释掉）
# HTTPS_PROXY=http://127.0.0.1:7890
```

> **说明**：默认接的是 SiliconFlow 的 DeepSeek-V4-Flash，如需更换模型见第 8 节。

### 2.2 安装前端依赖（仅首次）

```bash
cd ~/Desktop/FDE/FR交接Agent/fr-agent/web
npm install
```

## 3. 启动方式

每次使用前需要同时启动**后端**和**前端**，打开两个终端窗口分别运行：

**终端 1 — 后端（FastAPI）**

```bash
cd ~/Desktop/FDE/FR交接Agent/fr-agent
.venv/bin/python3 -m uvicorn api.main:app --reload --port 8000
```

看到以下输出说明后端启动成功：

```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

**终端 2 — 前端（React）**

```bash
cd ~/Desktop/FDE/FR交接Agent/fr-agent/web
npm run dev
```

看到以下输出说明前端启动成功：

```
  VITE v6.x  ready in xxx ms
  ➜  Local:   http://localhost:5173/
```

**打开浏览器访问：`http://localhost:5173`**

> 注意：两个终端都要保持运行，关闭任意一个会导致页面无法使用。

## 4. 使用流程

页面顶部有四步工作流进度条，按顺序操作即可：

```
① 上传 CPT  →  ② 数据库增强（可选）  →  ③ AI 分析  →  ④ 查看 & 导出
```

### 第一步：上传 CPT 文件

- 点击页面中央的上传区域，选择 `.cpt` 文件
- 或将文件直接拖拽到上传区域
- 上传成功后自动解析，显示报表基本信息（FR版本、Sheet数、数据集数等）

### 第二步：数据库增强（可选）

- 仅当报表使用**实时数据库查询**（DBTableData 类型数据集）时，左侧侧边栏会出现数据库配置入口
- 如果报表使用内嵌数据集（EmbeddedTableData），侧边栏会直接显示"无需数据库连接"，跳过此步
- 配置方式见第 5.1 节
- **此步可以跳过**，跳过后 AI 分析仍可正常运行，但字段语义分析质量会略低

### 第三步：AI 分析

- 点击蓝色"**启动 AI 分析**"按钮
- 分析过程约 15–60 秒（取决于报表复杂度）
- 等待期间会实时显示分析进度日志

### 第四步：查看结果 & 导出

- 分析完成后，结果以 7 个 Tab 展示（见第 5 节）
- 在"导出"Tab 下载 Markdown 或 HTML 交接文档

## 5. 各功能说明

### 5.1 数据库增强（左侧侧边栏）

**作用**：接入报表对应的 MySQL 数据库，获取每张表的字段类型和字段注释，让 AI 能理解字段的实际业务含义。

**操作步骤**：

1. 填写 FineReport WEB-INF 路径（用于自动读取连接配置）：
   ```
   默认路径示例：/Users/你的用户名/FineReport/webapps/webroot/WEB-INF
   ```
2. 在"密码"输入框填写对应数据库连接的密码
3. 点击"🔗 连接并获取字段"
4. 看到"✅ 数据库字段已成功接入"表示成功

> **注意**：密码只在本地使用，不会上传到任何服务器。  
> FR 的连接密码是 RSA 加密存储，系统无法自动读取，需手动输入。

---

### 5.2 概览 Tab

分析完成后首先展示，包含：

- **报表业务用途**：一句话概括这张报表是做什么的
- **布局结构**：表单型/列表型/分组展开，哪行是表头、数据行、合计行
- **数据集关系**：各数据集是主表还是参数表，如何关联
- **关键字段含义**：重要字段的业务语义（可展开查看）
- **风险点与注意事项**：AI 识别的潜在问题

---

### 5.3 交互链路 Tab

展示每个参数控件如何驱动数据流动：

| 字段 | 说明 |
|------|------|
| 作用 | 该控件在筛选/驱动数据中的作用 |
| SQL | 如何影响 SQL 查询（参数变量、动态条件） |
| 展示 | 哪些单元格/字段展示了来自该数据集的数据 |
| 原因 | 为什么这样设计（业务角度解释） |

底部可展开查看"参数控件汇总表"（控件名/类型/数据来源/字段映射）。

---

### 5.4 指标字典 Tab

列出报表中所有有业务意义的指标和字段，包含：

- 指标名（优先从标签单元格提取真实业务名称）
- 来源字段（数据库字段名或单元格坐标）
- 数据集、类型（维度/度量）、单位
- 公式（公式类指标）
- 业务含义

可展开查看"公式详细解释"，逐个说明报表中有意义的公式含义。

---

### 5.5 开发步骤 Tab

AI 推断的从零复现该报表的开发步骤，格式为时间线列表。

> 内容由 AI 推断，请核实后参考使用。填报报表会包含填报属性配置步骤。

底部还会显示（如果有）：
- **条件高亮规则**：每条规则的触发条件、颜色、影响字段
- **填报提交配置**：写入哪张表、主键字段、字段映射关系

---

### 5.6 数据血缘 Tab

可视化展示"控件 → SQL参数 → 数据集 → 展示字段"的完整数据流向图。

节点颜色含义：

| 颜色 | 含义 |
|------|------|
| 🔵 蓝色 | 参数控件 |
| 🟢 绿色 | SQL 数据集 |
| 🟠 橙色 | 展示字段 |
| ⚪ 灰色 | 控件选项数据集 |

如果出现"未找到 SQL 参数连接的控件"警告，说明这些控件可能通过单元格过滤条件或前端 JS 实现筛选，需要人工核实。

底部可展开查看每个数据集的详细信息（字段列表、SQL 语句）。

---

### 5.7 问答 Tab

基于报表结构和 AI 分析结论，支持自然语言追问。

**示例问题**：
- "这张报表有哪些参数控件，分别控制什么数据？"
- "grade 字段代表什么业务含义？"
- "这张报表接手后，最需要注意哪些地方？"
- "如果要新增一个科目筛选条件，应该怎么改？"

支持多轮连续对话，上下文自动保留。

---

### 5.8 导出 Tab

| 格式 | 适用场景 | 操作 |
|------|----------|------|
| **Markdown** | 粘贴到 Confluence / Notion / 飞书文档 | 点击"👁️ 预览"查看，点击"⬇️ 下载 MD"保存 |
| **HTML** | 直接发给接手方，或打印为 PDF | 点击"⚙️ 生成预览"后可下载，`Ctrl+P` 打印为 PDF |

## 6. 导出文档

交接文档包含以下 10 个标准章节：

```
一、报表概述          业务用途 + 布局结构
二、基本信息          文件名 / FR版本 / Sheet数 / 文件大小
三、数据来源与字段关系  血缘图 + 数据集清单 + SQL 语句 + 指标字典
四、交互链路          控件→参数→数据→展示完整链路
五、单元格展示        字段对照表 + 绑定明细 + 公式解释
六、从零复现步骤       AI 推断的开发步骤
七、条件高亮规则       业务规则（颜色/触发条件/影响字段）
八、填报配置          写入表/主键/字段映射（仅填报报表）
九、注意事项与风险点   AI 识别的潜在问题
十、交接确认          交接方 & 接收方签字确认表格
```

> ⚠️ 文档由 AI 自动推断生成，请交接双方核实确认后使用。

## 7. 常见问题

### Q：页面打开是空白或报错

检查两个终端是否都在运行：
- 后端：终端 1 应显示 `Uvicorn running on http://127.0.0.1:8000`
- 前端：终端 2 应显示 `Local: http://localhost:5173/`

### Q：AI 分析报错 "Connection error"

原因是 LLM API 无法访问。检查：
1. `.env` 文件中 `SILICONFLOW_API_KEY` 是否正确
2. `HTTPS_PROXY` 是否指向了未运行的代理，如果不需要代理，把这行注释掉：
   ```env
   # HTTPS_PROXY=http://127.0.0.1:7890
   ```
3. 网络是否能访问 `https://api.siliconflow.cn`

### Q：数据库连接失败

常见原因：
- 密码输入错误
- MySQL 未允许本地连接（检查 `bind-address` 配置）
- 端口号不对（默认 3306）
- 防火墙限制

> 数据库增强是可选功能，连接失败不影响 AI 分析，可以跳过。

### Q：上传 CPT 文件后显示解析错误

可能原因：
- 文件不是标准 `.cpt` 格式（FR 特有格式，非通用 CPT）
- FineReport 版本过旧（目前支持 9.0 / 10.0 / 11.x）

### Q：血缘图加载很慢

首次切换到"数据血缘" Tab 时需要向后端请求并用 Mermaid.js 渲染，通常 2–5 秒。刷新 Tab 或重新切换即可。

### Q：想重新上传另一个报表文件

点击左侧侧边栏底部的"↩️ 重新上传文件"按钮，确认后清空当前数据，回到上传界面。

## 8. LLM 模型切换

只需修改 `.env` 文件，无需改动代码：

**切换到本地 Ollama**

```env
SILICONFLOW_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:14b
SILICONFLOW_API_KEY=ollama
```

**切换到 OpenAI GPT-4o**

```env
SILICONFLOW_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o
SILICONFLOW_API_KEY=sk-你的OpenAI密钥
```

**切换到公司内网 AI 网关**

```env
SILICONFLOW_BASE_URL=https://it-ai.你公司域名.com/
LLM_MODEL=claude-sonnet-4-6
SILICONFLOW_API_KEY=sk-内网密钥
```

修改 `.env` 后重启后端（终端 1 按 `Ctrl+C` 再重新运行 uvicorn 命令）即可生效。
