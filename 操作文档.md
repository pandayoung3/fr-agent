# FR-Agent 操作文档

版本：React + FastAPI v2
适用对象：使用 FR-Agent 做 FineReport 报表交接的开发人员、实施人员和维护人员

## 1. 使用前准备

### 环境要求

| 依赖 | 建议版本 | 说明 |
| --- | --- | --- |
| Python | 3.9+ | 后端和核心解析模块 |
| Node.js | 18+ | React 前端开发服务 |
| npm | 9+ | 前端依赖管理 |
| MySQL / MariaDB | 可选 | 用于字段元数据增强 |
| FineReport | 可选 | 用于读取 WEB-INF 连接配置 |

### 配置 LLM

复制 `.env.example` 为 `.env`，填写：

```env
SILICONFLOW_API_KEY=sk-your-key
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
LLM_MODEL=deepseek-ai/DeepSeek-V4-Flash

# 如需代理
# HTTPS_PROXY=http://127.0.0.1:7890
```

支持任意 OpenAI 兼容端点，例如 OpenAI、SiliconFlow、Ollama 或企业内网网关。

## 2. 启动方式

需要同时启动后端和前端。

### 终端 1：后端

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m uvicorn api.main:app --reload --port 8000
```

看到以下信息表示启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
```

### 终端 2：前端

```bash
cd web
npm install
npm run dev
```

打开：

```text
http://localhost:5173
```

## 3. 标准使用流程

```text
上传 CPT
-> 可选数据库增强
-> 启动 AI 分析
-> 查看结果
-> 问答追问
-> 导出 Markdown / HTML
```

### 3.1 上传 CPT

- 选择或拖拽 `.cpt` 文件。
- 系统调用 `/api/parse`。
- 成功后展示 FR 版本、Sheet 数、数据集、控件、单元格绑定等结构信息。

### 3.2 数据库增强（可选）

当报表含 DBTableData 时，可填写 FineReport WEB-INF 路径并输入数据库密码。

系统只读取字段元数据：

- 字段名
- 字段类型
- 字段注释

系统不读取真实业务数据，不自动解密 FineReport 密码。

数据库增强失败不会阻塞 AI 分析，但字段语义质量可能下降。

### 3.3 AI 分析

点击启动分析后，系统通过 SSE 返回进度。分析结果包括：

- 报表用途
- 布局说明
- 数据集关系
- 字段语义
- 交互链路
- 公式解释
- 条件高亮含义
- 复现步骤
- 指标字典
- 风险点

### 3.4 查看结果

React 前端当前按 Tab 展示：

- 概览
- 交互链路
- 指标字典
- 开发步骤
- 数据血缘
- 问答
- 导出

### 3.5 导出

支持：

- Markdown：适合粘贴到飞书、Notion、Confluence。
- HTML：适合直接发送、归档或打印为 PDF。

导出文档由 AI 辅助生成，必须由交接双方人工核实。

## 4. 常见问题

### 页面打不开

检查：

- 后端是否运行在 `http://127.0.0.1:8000`
- 前端是否运行在 `http://localhost:5173`
- 前端代理或 API 路径是否正确

### AI 分析失败

检查：

- `.env` 中 API key 是否正确。
- `SILICONFLOW_BASE_URL` 是否可访问。
- 代理配置是否指向正在运行的代理。
- 模型是否支持 JSON 输出。

### 数据库增强失败

常见原因：

- 密码错误。
- MySQL 不允许本地连接。
- 端口或 host 错误。
- WEB-INF 路径不正确。
- 当前数据库类型不是 MySQL / MariaDB。

数据库增强是可选能力，可跳过。

### 解析失败

可能原因：

- 文件不是 FineReport `.cpt`。
- 文件版本或结构超出当前 Parser 支持范围。
- CPT 内部 XML 缺失关键节点。

解析失败时，应保留错误信息并进入 Parser 任务排查。

## 5. 安全边界

- 数据库密码只用于本地连接，不应写入日志、导出文档或 Git。
- 当前设计只读取元数据，不读取业务数据。
- AI 输出必须人工复核。
- 生成文档可能包含字段名、SQL 和业务语义，应按团队安全要求传递。
