# PRD：FR-Agent

版本：0.1
状态：讨论稿，等待用户检验后定稿

## 1. 产品定位

FR-Agent 是一个 FineReport 报表交接工具。用户上传 `.cpt` 文件后，系统自动解析报表结构，结合可选数据库字段元数据和 LLM 语义分析，生成可核实、可交付的 Markdown / HTML 交接文档，并支持围绕报表上下文追问。

产品分两层：

- 产品层：给开发/实施/维护人员使用的本地 Web 工具。
- 能力层：未来可被 Agent 平台调用的报表解析、分析、问答和文档生成服务。

## 2. 目标用户

- 接手 FineReport 报表维护的开发人员。
- 负责报表迁移、重构、交接、验收的技术负责人。
- 需要批量理解旧报表结构的实施或数据团队。

## 3. P0 主线决策

- 主线框架：React + TypeScript + Vite + FastAPI + Python 核心模块。
- 交付形态：本地开发者工具。
- Streamlit：历史 MVP 入口，后续尽快下线或归档，不再新增功能。
- 平台嫁接：P2，优先考虑 MCP / 插件，不抢 P0 资源。
- 评分标准：P0 已定稿 100 分交接可用性评分标准；自动评分实现不进入 P0。

## 4. 当前已具备能力

- 上传并解析 `.cpt`。
- 提取数据集、SQL、参数控件、单元格绑定、公式、条件高亮、填报配置。
- 可选接入 MySQL 字段元数据。
- 生成数据血缘图。
- LLM 输出报表用途、字段语义、交互链路、指标字典、复现步骤和风险点。
- 支持报表问答。
- 导出 Markdown / HTML 交接文档。
- React 前端和 FastAPI 后端已存在。

## 5. P0 稳定化范围

P0 不做新功能，目标是让现有主线稳定、可运行、可验证、可交接。

必须完成：

- README 和 `操作文档.md` 统一到 React + FastAPI v2 主线。
- 补齐后端依赖。
- 明确前后端启动命令。
- 建立 API contract 文档。
- 建立固定样例验收流程。
- 明确 Streamlit 下线策略。
- 将 SOP、Handoff、需求池纳入真实项目文档体系。
- P0 最后定稿评分标准，用于初步检验当前产品效果。

## 6. P1 候选

- Parser 自动化测试。
- API smoke test。
- 前端 build/lint 固化。
- 批量 CPT 解析。
- 分析历史和导出记录。
- 多模型配置体验优化。
- 评分系统 MVP 自动化实现。

## 7. P2 候选

- MCP Server / Agent 平台插件。
- 报表版本差异对比。
- 团队知识库发布。
- Docker / 内网部署。
- 多 Agent 产品化工作流。

## 8. 明确不做

- P0 不做多用户权限系统。
- P0 不做云端部署。
- P0 不自动解密 FineReport RSA 密码。
- 默认不读取真实业务数据，只读字段元数据。
- AI 输出必须保留人工核实环节。

## 9. 验收标准

- 用户能按文档启动后端和前端。
- 上传样例 CPT 后能得到 parsed 结果。
- 可选数据库增强失败时不阻塞 AI 分析。
- AI 分析完成后能查看结果 Tab。
- Markdown / HTML 导出可用。
- 生成文档明确提示需要人工核实。
- 每个 P0 任务有验证命令或人工验收路径。
- P0 结束前形成一版 100 分评分标准，用于评估解析和交接内容是否真实可用。
